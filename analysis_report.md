# Анализ проблем конфигурации сборки OpenClaw Desktop

## Текущий стек
- Electron 40.8.0
- electron-builder 24.6.4 (установлена версия 26.8.1)
- React 18 + TypeScript + Vite + Tailwind CSS
- Интеграции: OpenClaw JS, Google APIs, GitHub, Perplexity API

## Выявленные проблемы

### 1. Несоответствие версий electron-builder
- В `package.json` указана версия `24.6.4`, но локально установлена `26.8.1`.
- Это приводит к конфликту схемы конфигурации, так как версия 26.x имеет изменения в допустимых полях.

### 2. Неизвестное свойство 'main'
- В корне `package.json` присутствует поле `"main": "main.js"`.
- electron-builder 26.8.1 интерпретирует его как часть конфигурации и сообщает об ошибке, поскольку свойство `main` недопустимо в корне конфигурации.
- Свойство `main` необходимо для Electron (точка входа), но должно быть исключено из конфигурации electron‑builder.

### 3. Неизвестные свойства в linux.desktop
- В конфигурации присутствует секция `linux.desktop` со свойствами `Name`, `Comment`, `StartupWMClass`, `MimeType`, `Categories`.
- В текущем `electron-builder.json` эти свойства отсутствуют, возможно они были унаследованы из старой конфигурации или находятся в другом файле.
- В схеме electron‑builder 26.x данные свойства, вероятно, перемещены или удалены.

### 4. Неправильный тип mac.notarize
- В конфигурации указано `mac.notarize` (не в текущем файле), которое должно быть булевым значением.
- Скорее всего, используется устаревший формат нотаризации.

### 5. Неизвестные свойства в win
- В конфигурации присутствуют `publisherName`, `signingHashAlgorithms`, `certificateFile`, `certificatePassword`, `rfc3161TimeStampServer`.
- Эти свойства отсутствуют в текущем `electron-builder.json`. Возможно, они были добавлены для подписи Windows и требуют обновления формата.

### 6. Отсутствие зависимости electron-notarize
- В `package.json` нет пакета `electron-notarize`, который используется в скрипте `build/notarize.js`.

### 7. Устаревшая структура конфигурации
- Конфигурация использует поле `"build": { "extends": "./electron-builder.json" }` в `package.json`.
- В electron‑builder 26.x рекомендуется использовать отдельный файл конфигурации (`electron-builder.yml` или `electron-builder.json`) без поля `build`.

## Возможные причины
- Обновление electron‑builder без корректировки конфигурации.
- Наличие остаточных настроек от предыдущих версий.
- Использование примеров конфигурации, несовместимых с актуальной схемой.

## Результат
Сборка прерывается на этапе валидации конфигурации, что делает невозможным создание дистрибутивов для Windows, macOS и Linux.
