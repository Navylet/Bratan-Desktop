# Отчёт об анализе неудачной сборки OpenClaw Desktop

**Дата:** 2026-03-08  
**Репозиторий:** Navylet/Bratan-Desktop  
**Workflow run:** [Build and Release](https://github.com/Navylet/Bratan-Desktop/actions/runs/11181673234) (ID: 11181673234)  
**Анализированный run:** 22828211446 (последний failed run, связанный с обновлением electron-builder)

## Причина сбоя

Сборка падает на шаге `npm run dist:win-builder` (Windows job) с ошибкой:

```
gyp: binding.gyp not found (cwd: D:\a\Bratan-Desktop\Bratan-Desktop) while trying to load binding.gyp
```

**Корневая причина:**  
В файле `electron-builder.json` были включены опции `npmRebuild: true` и `nodeGypRebuild: true`, которые заставляют electron-builder пытаться пересобрать нативные модули для Electron. Поскольку в проекте отсутствует файл `binding.gyp` (нет нативных модулей, требующих сборки), процесс `node-gyp` завершается с ошибкой.

**Дополнительные факторы риска:**

1. **Отсутствие иконок** – в конфиге указаны пути `build/icon.ico`, `build/icon.png`, `build/icon.icns`, но папка `build` пуста либо отсутствует.
2. **Скачивание Chromium** – зависимость `puppeteer` может пытаться загрузить Chromium при установке зависимостей, что увеличивает время сборки и может приводить к сбоям.

## Выполненные исправления

### 1. Конфигурация electron-builder (`electron-builder.json`)

- Установлены `npmRebuild: false`, `nodeGypRebuild: false`, `skipNodeRebuild: true` – отключают попытку пересборки нативных модулей.
- Добавлено глобальное поле `icon: "assets/icon.png"` – позволяет electron-builder автоматически сгенерировать иконки для всех платформ из одного PNG-файла.
- Удалены специфические поля `icon` внутри секций `win`, `linux`, `mac`.

### 2. Workflow (`build.yml`)

- Добавлена переменная окружения `PUPPETEER_SKIP_DOWNLOAD: true` на уровне каждого job (Linux, Windows, macOS) – предотвращает скачивание Chromium через puppeteer.

### 3. Структура проекта

- Создана папка `build/` (пока пустая). Иконки будут сгенерированы electron-builder во время сборки.

## План дальнейших действий

### Немедленные шаги

1. **Запустить workflow с внесёнными изменениями** – сделать push в ветку `main` или запустить workflow вручную через GitHub Actions.
2. **Мониторинг логов** – убедиться, что ошибка `binding.gyp not found` больше не появляется, и сборка проходит до конца.

### Если сборка всё ещё падает

- **Проверить наличие иконок** – если electron-builder не смог сгенерировать иконки, можно создать placeholder-файлы вручную:
  ```bash
  cp assets/icon.png build/icon.png
  cp assets/icon.png build/icon.ico   # для Windows (лучше сконвертировать в ICO)
  cp assets/icon.png build/icon.icns  # для macOS (лучше сконвертировать в ICNS)
  ```
- **Установить Windows Build Tools** – добавить в Windows job шаг установки инструментов для сборки нативных модулей (на всякий случай):
  ```yaml
  - name: Install Windows build tools
    run: npm install --global windows-build-tools
  ```
  (Использовать только если появятся ошибки компиляции других нативных зависимостей.)

### Долгосрочные улучшения

- **Вынести иконки в отдельный набор** – подготовить профессиональные иконки для всех платформ (ICO, ICNS, PNG разных размеров).
- **Оптимизировать зависимости** – рассмотреть возможность замены `puppeteer` на `puppeteer-core` или исключения его из production-сборки.
- **Настроить кэширование** – добавить кэширование для `node_modules` и `~/.cache/electron-builder` (уже частично есть).

## Риски

- Изменение `npmRebuild` на `false` может привести к проблемам, если в будущем появятся нативные зависимости, требующие сборки под Electron. В таком случае нужно будет либо включить пересборку, либо предварительно собирать нативные модули в отдельном шаге.
- Использование одного PNG-файла для всех платформ может дать неидеальный результат (особенно для маленьких иконок в системных треях). Рекомендуется подготовить отдельные наборы иконок.

## Заключение

Основная причина сбоя устранена. Внесённые изменения должны позволить сборке завершиться успешно. После успешного прохождения workflow можно будет приступить к тестированию собранных артефактов и подготовке релиза.
