# OpenClaw Desktop - Тест-план

## Цель

Комплексное тестирование Electron‑приложения OpenClaw Desktop на трёх платформах (Windows, macOS, Linux) для обеспечения качества, выявления дефектов и подтверждения соответствия требованиям.

## Область тестирования

- Функциональность всех 7 вкладок (Чат, Логи, Файлы, Редактор, Агенты, Интеграции, Настройки)
- Интеграции с внешними сервисами (OpenClaw JS, Google APIs, GitHub, Perplexity)
- Кросс‑платформенная совместимость (Windows 10/11, macOS latest, Ubuntu/Debian/Fedora)
- UI/UX и accessibility
- Обработка ошибок и edge cases
- Производительность и стабильность

## Подход

- Ручное тестирование (exploratory + по тест‑кейсам)
- Статический анализ кода (review)
- Автоматизация (smoke‑тесты, если будут написаны)
- Тестирование в изолированном окружении (WSL2/Linux)

## Критерии качества

- Все ключевые сценарии пользователя протестированы
- Критические баги (critical) отсутствуют
- Баг‑репорты содержат environment, шаги воспроизведения, expected/actual, скриншоты
- Приоритизация по severity: critical, major, minor, trivial

## Тест‑кейсы

### 1. Вкладка "Чат"

**Цель:** Проверить базовую функциональность чата.

- **TC‑CHAT‑001:** Отправка сообщения → появляется в истории
- **TC‑CHAT‑002:** Получение ответа от агента → отображается в чате
- **TC‑CHAT‑003:** Очистка истории → все сообщения удаляются
- **TC‑CHAT‑004:** Ошибка сети → показывается уведомление, сообщение не теряется
- **TC‑CHAT‑005:** Поддержка специальных символов и эмодзи
- **TC‑CHAT‑006:** Автопрокрутка при добавлении новых сообщений
- **TC‑CHAT‑007:** Поллинг сообщений от OpenClaw JS (авто‑добавление)

### 2. Вкладка "Логи"

**Цель:** Проверить просмотр и фильтрацию логов.

- **TC‑LOG‑001:** Загрузка логов → отображаются записи
- **TC‑LOG‑002:** Фильтрация по уровню (error, warn, info) → показываются только выбранные
- **TC‑LOG‑003:** Поиск по тексту → находит записи
- **TC‑LOG‑004:** Экспорт логов → файл скачивается
- **TC‑LOG‑005:** Автообновление логов (если есть)
- **TC‑LOG‑006:** Очистка логов → список становится пустым

### 3. Вкладка "Файлы"

**Цель:** Проверить навигацию по файловой системе и операции с файлами.

- **TC‑FILES‑001:** Навигация по директориям → отображается содержимое
- **TC‑FILES‑002:** Открытие файла → открывается в редакторе
- **TC‑FILES‑003:** Создание новой папки → папка появляется в списке
- **TC‑FILES‑004:** Удаление файла → файл исчезает, запрашивается подтверждение
- **TC‑FILES‑005:** Переименование файла/папки
- **TC‑FILES‑006:** Копирование/перемещение файлов
- **TC‑FILES‑007:** Открытие внешней папки (системный файловый менеджер)
- **TC‑FILES‑008:** Обновление списка (refresh)

### 4. Вкладка "Редактор"

**Цель:** Проверить функциональность редактора кода.

- **TC‑EDITOR‑001:** Редактирование кода → изменения сохраняются
- **TC‑EDITOR‑002:** Подсветка синтаксиса → работает для разных языков
- **TC‑EDITOR‑003:** Поиск и замена → находит и заменяет текст
- **TC‑EDITOR‑004:** Множественные вкладки → можно открывать несколько файлов
- **TC‑EDITOR‑005:** Автоопределение языка по расширению
- **TC‑EDITOR‑006:** Сохранение файла (Ctrl+S, кнопка Save)
- **TC‑EDITOR‑007:** Закрытие файла с несохранёнными изменениями → предупреждение
- **TC‑EDITOR‑008:** Смена темы (light/dark)
- **TC‑EDITOR‑009:** Настройки редактора (размер шрифта, табуляция)

### 5. Вкладка "Агенты"

**Цель:** Проверить управление агентами.

- **TC‑AGENTS‑001:** Отображение списка агентов → показываются все агенты
- **TC‑AGENTS‑002:** Мониторинг статуса → статус обновляется в реальном времени
- **TC‑AGENTS‑003:** Запуск/остановка агента → команды выполняются
- **TC‑AGENTS‑004:** Просмотр логов агента → логи отображаются
- **TC‑AGENTS‑005:** Создание нового агента → появляется в списке
- **TC‑AGENTS‑006:** Удаление агента → удаляется из списка
- **TC‑AGENTS‑007:** Мониторинг задач (task monitor) → отображается очередь

### 6. Вкладка "Интеграции"

**Цель:** Проверить интеграции с внешними сервисами.

- **TC‑INTEG‑001:** Настройка Google OAuth → открывается окно авторизации
- **TC‑INTEG‑002:** Подключение GitHub → запрашивается токен
- **TC‑INTEG‑003:** Настройка Perplexity API → ключ сохраняется
- **TC‑INTEG‑004:** Отключение интеграции → интеграция деактивируется
- **TC‑INTEG‑005:** Google Drive: загрузка списка файлов
- **TC‑INTEG‑006:** GitHub: загрузка списка репозиториев
- **TC‑INTEG‑007:** Perplexity: выполнение поискового запроса → результат в чате
- **TC‑INTEG‑008:** Обработка ошибок (неверный токен, сетевые сбои)

### 7. Вкладка "Настройки"

**Цель:** Проверить настройки приложения.

- **TC‑SETTINGS‑001:** Изменение темы (светлая/тёмная) → интерфейс меняется
- **TC‑SETTINGS‑002:** Настройка горячих клавиш → клавиши работают
- **TC‑SETTINGS‑003:** Изменение языка → интерфейс переключается
- **TC‑SETTINGS‑004:** Сброс настроек → возвращаются значения по умолчанию
- **TC‑SETTINGS‑005:** Настройка шрифта → применяется ко всему интерфейсу
- **TC‑SETTINGS‑006:** Настройка уведомлений (включение/выключение)
- **TC‑SETTINGS‑007:** Настройка автозапуска
- **TC‑SETTINGS‑008:** Настройка пути CLI (openclaw)
- **TC‑SETTINGS‑009:** Настройка порта Gateway

## Smoke‑тест (быстрая проверка сборки)

1. Запустить приложение
2. Проверить, что все 7 вкладок отображаются
3. Отправить тестовое сообщение в чат
4. Открыть вкладку «Файлы» — виден список
5. Перейти в «Настройки» — сбросить настройки
6. Проверить статус Gateway
7. Закрыть приложение

## Edge Cases & Обработка ошибок

- **Сеть:** Отключение интернета → graceful degradation
- **Файлы:** Попытка открыть бинарный файл → сообщение об ошибке
- **Файлы:** Нет прав на запись → уведомление
- **Интеграции:** Неверный токен → понятная ошибка
- **Интеграции:** Сервис недоступен → retry/timeout
- **Память:** Большие файлы (например, 100 МБ) → индикатор прогресса
- **Окно:** Закрытие во время операции → предупреждение
- **Окно:** Минимизация во время долгой задачи → работа продолжается
